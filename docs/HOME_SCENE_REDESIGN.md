# 家庭场景重设计方案

> 版本：v2.0 重设计
> 目标：从「方块拼图」升级为「真实俯视角家居」

---

## 一、当前问题

| 问题 | 现状 | 目标 |
|------|------|------|
| 视觉风格 | 彩色方块 + 网格线 | 真实房间平面图（木地板/瓷砖/墙壁/门窗）|
| 房间布局 | 三个房间挤在一个画面 | 每次只显示一个房间，左右切换 |
| 家具布置 | 硬编码固定位置 | 拖拽自由放置，持久化存储 |
| 宠物行为 | 随机游走 + 状态驱动走向方块 | 有意图的路径：找食盆/床/玩具，避开墙壁 |
| 购买系统 | 无 | 商店购买家具，解锁新物品 |

## 二、房间切换系统

### 2.1 房间列表（初始 3 间，可扩展）

```
客厅  ←→  卧室  ←→  厨房
```

- 每个房间是独立的 Phaser Scene（或同一 Scene 内切换地图层）
- 切换动画：左右滑动过渡（300ms ease）
- 底部/顶部显示房间标签导航栏，点击直接跳转
- 每个房间有独立的家具布置数据（已有后端支持：`placed_furniture` 表有 `room` 字段扩展空间）

### 2.2 房间切换实现方式

**方案：同一 Phaser Scene，切换地图层**
- 一个 Scene，维护 `currentRoom: 'living' | 'bedroom' | 'kitchen'`
- 切换时：旧房间内容 tween 滑出，新房间内容 tween 滑入
- 宠物跟随当前房间显示（宠物有「当前所在房间」状态）
- 优点：无重建 Game 开销，动画流畅

## 三、真实视觉风格

### 3.1 地板

| 房间 | 地板材质 | 颜色 |
|------|---------|------|
| 客厅 | 木地板（横向条纹 + 接缝） | 暖棕 #c8956c |
| 卧室 | 地毯（纯色 + 细纹理） | 米白 #f0e8d8 |
| 厨房 | 瓷砖（方格 + 缝隙） | 浅灰 #e8e8e8 |

实现：Phaser Graphics 绘制，不依赖图片资源

### 3.2 墙壁与门

- 四周绘制厚墙（16px）
- 墙面颜色：米白 #f5f0e8，带轻微阴影
- 每个房间至少一个门洞（矩形缺口），门洞处绘制门框
- 相邻房间共享门洞位置（保持空间感一致）

### 3.3 窗户

- 外墙绘制窗户（矩形 + 内框 + 浅蓝透明玻璃）
- 客厅：南墙两扇窗
- 卧室：东墙一扇窗
- 厨房：北墙一扇窗

### 3.4 家具视觉（不依赖图片）

用 Phaser Graphics 组合绘制，每种家具有专属形状：

| 家具 | 视觉描述 |
|------|----------|
| 沙发 | 深棕矩形 + 靠背（后侧细长条）+ 两侧扶手 |
| 床 | 矩形床架 + 床头板（上侧加厚）+ 被子（内层浅色矩形）|
| 食盆 | 小圆形 + 橙色内圈 |
| 水盆 | 小圆形 + 蓝色内圈 |
| 玩具球 | 圆形 + 对角线分色 |
| 猫爬架 | 竖向矩形 + 横向平台 |
| 桌子 | 矩形 + 四角小方块（桌腿）|
| 植物 | 圆形绿叶 + 棕色花盆 |

## 四、家具自由布置系统

### 4.1 编辑模式 vs 生活模式

- **生活模式**（默认）：宠物正常活动，家具不可移动，点击家具触发互动
- **编辑模式**（点击「布置」按钮进入）：
  - 家具可拖拽移动
  - 拖拽时显示半透明预览 + 网格吸附（TILE_SIZE = 32px）
  - 碰撞检测：家具之间不能重叠，不能放在墙内
  - 退出编辑模式时自动保存

### 4.2 家具操作

| 操作 | 触发方式 |
|------|----------|
| 移动 | 编辑模式下拖拽 |
| 旋转 | 编辑模式下双击（0°/90°/180°/270°）|
| 删除 | 编辑模式下点击后出现删除按钮 |
| 放置新家具 | 从「家具库」面板拖入场景 |

### 4.3 家具库面板

- 编辑模式下右侧滑出抽屉
- 显示所有已购买但未放置的家具
- 点击即放置到房间中央，再拖拽到目标位置

### 4.4 后端已有支持

- `furniture_templates` 表：家具类型定义
- `placed_furniture` 表：`tile_x / tile_y / flipped`
- 需扩展：`room` 字段（客厅/卧室/厨房），`rotation` 字段（0/90/180/270）

## 五、购买家具系统

### 5.1 货币系统

- 新增 `coins` 字段到 `User` 表
- 获取途径：登录奖励、宠物互动、好友社交
- 初始赠送：注册送 500 金币

### 5.2 商店页面 `/shop`

- 展示所有可购买家具模板
- 每件家具有价格、预览图（同场景绘制风格）
- 购买后加入用户的「家具库存」（新增 `user_furniture_inventory` 表）
- 基础必需品（食盆/水盆/床/玩具）默认赠送，无需购买

### 5.3 数据模型扩展

```python
class UserFurnitureInventory(Base):
    __tablename__ = "user_furniture_inventory"
    id: int
    user_id: int          # FK → User
    template_id: int      # FK → FurnitureTemplate
    quantity: int         # 拥有数量
    purchased_at: datetime
```

## 六、宠物行为真实化

### 6.1 行为状态机（升级版）

| 状态 | 触发条件 | 行为描述 |
|------|----------|----------|
| `wandering` | 默认/无需求 | 在当前房间随机游走，走到墙边会转向 |
| `seeking_food` | fullness < 40 | 寻找食盆，A* 路径规划到达 |
| `seeking_water` | hydration < 40 | 寻找水盆 |
| `seeking_rest` | energy < 30 | 寻找床，到达后进入睡眠动画 |
| `playing` | affection < 50，有玩具 | 找玩具，到达后原地弹跳 |
| `grooming` | cleanliness < 30 | 原地梳理毛发动画（小圈圈）|
| `social` | 有其他宠物在同房间 | 走向对方，停在旁边 |
| `sleeping` | energy < 10 | 趴在床上，不响应 wandering |

### 6.2 路径规划

- 实现简单 A* 算法（20×20 网格）
- 墙壁、已放置家具标记为不可通行
- 目标点：最近的符合需求的家具
- 路径平滑：沿路径点用 Tween 连续移动，而非直线闪现

### 6.3 动画细节

| 行为 | 动画 |
|------|------|
| 行走 | 轻微左右摇摆（scaleX 0.95~1.05）|
| 吃东西 | 到达食盆后低头动作 |
| 睡觉 | 缩小到 0.85，添加「zzz」浮动文字 |
| 玩耍 | 小跳跃动画（y 上下 ±8px）|
| 梳理 | 原地旋转 ±15° 摇摆 |
| 开心 | 偶发向上弹出「♥」emoji |

### 6.4 多宠物交互

- 同房间内两只宠物靠近时（< 3 格）触发问候动画
- 好友关系：靠近时概率触发玩耍状态

## 八、技术注意事项

1. **不引入图片资源**：所有视觉用 Phaser Graphics 纯代码绘制，保持零静态资源依赖
2. **性能**：A* 只在目标改变时重算，不每帧重算；家具碰撞图用二维数组缓存
3. **向后兼容**：Phase A 完成前，旧的 `HOME_SCENE_OBJECTS` 作为兜底保留，不删除
4. **移动端**：TILE_SIZE 根据屏幕宽度自适应（最小 24px，最大 36px）
5. **多宠物**：每只宠物独立状态机，互不干扰，但可感知彼此位置
